# __tests__/routes

ルートハンドラの統合テストを配置するディレクトリ。

## テストの対象

- 各エンドポイントの正常系・異常系（認証エラー・権限エラーなど）
- ミドルウェアとuseCaseが連携して正しく動作すること

## バリデーションテストはここでは行わない

リクエストボディのバリデーション（必須項目・フォーマット・文字数など）のテストはここでは書かない。
バリデーションのテストは `__tests__/validators/` で行う。

```
# ここでやること
- 正常なリクエストで期待するレスポンスが返ること
- 認証なし → 401
- 管理者でないユーザー → 403
- 対象が存在しない → 404 / エラー

# ここでやらないこと
- 必須フィールドが欠けている → 422
- メールアドレス形式が不正 → 422
- パスワードが短すぎる → 422
```

## 新規ファイル追加時のチェックリスト

- [ ] ファイル名が対応するルートファイル名（`auth.test.ts` など）になっている
- [ ] `beforeEach` / `afterEach` でDBの状態をリセットしている
- [ ] バリデーションのテストを含んでいない
