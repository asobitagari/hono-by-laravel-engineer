# useCases/auth

認証に関するユースケースを配置するディレクトリ。

> 共通規約は [../README.md](../README.md) を参照。

## 配置すべきファイル一覧

| ファイル名 | 概要 |
|---|---|
| `loginUseCase.ts` | メールアドレスとパスワードで認証し、アクセストークン（JWT）とリフレッシュトークンを返す。リフレッシュトークンはDBに保存される |
| `logoutUseCase.ts` | リフレッシュトークンをDBから削除して無効化する |
| `refreshTokenUseCase.ts` | リフレッシュトークンをDB検証し、新しいアクセストークンとローテーション済みリフレッシュトークンを返す |
| `registerUseCase.ts` | 新規ユーザーを登録し、トークンを返す |
| `resetPasswordUseCase.ts` | パスワードリセットを実行する |

## 追加のチェック項目

- [ ] 認証・認可ドメインのロジックである
- [ ] 他ドメインのロジックを含んでいない（→ 該当ドメインの `useCases/` へ）
