# useCases

ビジネスロジックを担うユースケースを配置するディレクトリ。
ドメインごとにサブディレクトリを切る。

## 共通の責務

- HTTPの知識を持たない（`Context` や `Request` に依存しない）
- バリデーションは行わない（呼び出し元の route 層で実施済みとみなす）
- DBアクセスは直接行わず、`domain/` のリポジトリインターフェース経由で行う
- 第1引数にリポジトリインターフェースを受け取る（依存性の注入）

## 命名規則

```
{動詞}{対象}UseCase.ts
```

## 関数シグネチャの規約

- `export async function {name}UseCase(repo: XxxRepository, ...)` という名前付きエクスポートを1ファイルに1つ
- 第1引数はリポジトリインターフェース（`domain/` で定義）
- 戻り値は `Promise<T>` とし、型は同ファイル内に定義する
- エラーは例外をスローする（呼び出し元でハンドリング）

## 配置してはいけないもの（全ドメイン共通）

- Honoの `Context` や `Request` を引数に取る関数
- `zValidator` などのバリデーション処理
- DrizzleなどDB実装への直接依存（→ `infrastructure/repositories/` へ）

## 新規ファイル追加時のチェックリスト（全ドメイン共通）

- [ ] ファイル名が `{動詞}{対象}UseCase.ts` の形式になっている
- [ ] `export async function` の名前付きエクスポートが1つだけある
- [ ] 第1引数がリポジトリインターフェースになっている
- [ ] 引数に Hono の型（`Context` など）が含まれていない
- [ ] バリデーションロジックを含んでいない
- [ ] 戻り値の型が同ファイル内で定義されている
- [ ] エラーは戻り値ではなく例外としてスローしている
